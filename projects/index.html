项目

下面是我投入最多、也最能代表我能力的几个项目。

鲁班(原公司内部系统 · 架构经验)

这是我在大厂时期主导的核心内部系统——一套支撑营销活动的低代码平台。它把运营活动的研发成本从 4 人天降到接近 0,支撑过 400+ 场大型活动,峰值 10 万+ QPS,可用性 99.99%。

说明:这是公司内部系统,源代码归原公司所有,不公开。我在此沉淀的是架构经验与设计思路(多级缓存、SSR 降级、低代码渲染引擎、BFF 接入层等),不涉及任何公司代码。本博客所有相关文章讲的都是架构层面的设计取舍,可复用的方法论。

核心技术产出(经验沉淀):

  • 低代码架构:设计器 + 渲染引擎 + Schema 体系 + 多端运行时
  • 高并发方案:CDN → WAF → Nginx → SSR 兜底 → Node BFF → Redis 多级缓存
  • 稳定性保障:故障自动摘节点 + 全集群降级回源 CDN 静态页,P0 清零
  • 大促实战:11w QPS 故障复盘 → WAF 清洗 + 限流常态化

OpenLuban(个人开源 · 架构经验的开源实现)

与上面鲁班的关系:OpenLuban 是我离职后,把大厂鲁班沉淀的架构经验用全新代码重新实现的开源项目。架构同源,代码完全重写,无任何公司代码。

它不是一个单仓库,而是一个生态(均在 GitHub 开源):

子项目它做什么
luban低代码平台核心:设计器、渲染引擎、Schema 体系
luban-websiteSSR 站点,承载低代码页面并对外服务
luban-bffNode BFF 接入层,衔接前后端
luban-backendJava 版后端(Spring Boot)
luban-ui为低代码而生的 Vue 3 组件库
luban-ai-assistantAI 设计助手:用图片生成低代码页面
luban-architecture-design架构与产品设计文档

一个低代码平台从内核、SSR、BFF、Java 后端、组件库到 AI 助手都齐了——这是我架构能力和平台思维的集中体现。本博客技术实现类文章(渲染引擎、DDD、全栈指南等)均以 OpenLuban 开源代码为准。

多租户 SaaS 全栈平台(初创 · 从 0 到 1)

因涉及企业信息,这里只讲架构与方法,不展开业务细节。

我作为研发负责人,带 4 个人的小团队(加 AI 协同),2 个月从 0 到 1 交付了一个多租户 SaaS 平台,涵盖 C 端(uni-app 小程序)、运营后台(Vue 3 + Element Plus)、后端(Java Spring Boot)三端。

技术上,它解决了这些「硬」问题:

  • 多租户隔离:行级 tenant_id + 请求头标识 + SQL 审查,杜绝跨租户越权
  • RBAC 权限体系:权限 key 与种子数据对齐,过滤器链统一鉴权
  • 契约驱动:提交前用脚本守住「前端 URL ↔ 后端路由」「SQL ↔ 迁移 DDL」「权限 ↔ 种子」的边界
  • 数据库迁移治理:Flyway 版本号唯一性 + 多人协作规约
  • 可观测性:Prometheus 指标覆盖接口延迟、Token 消耗、调用成功率

但它最特别的地方不在技术,而在怎么做出来的:全程用 AI Agent 驱动开发——4 个人,做出了过去往往需要更大团队才能做出的系统。这也是这个博客「AI 研发实战」系列的真实背景。
