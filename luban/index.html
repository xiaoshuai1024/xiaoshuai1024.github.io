OpenLuban · 开源低代码营销平台

OpenLuban 是我把大厂「鲁班」沉淀的低代码架构经验，用全新代码重新实现的开源生态。架构同源，代码完全重写，不含任何公司代码。

它不是一个单仓库，而是一个 统一治理的 meta 仓：通过 git submodule 管理多个子项目，配套统一的 AI agent 规则、测试门禁与工作流。

4 应用服务

理解 OpenLuban，先抓住这条调用链。访客走 SSR 渲染端，运营走设计器，两者背后是同一套 BFF + 业务后端。

服务端口技术栈职责
web3000Nuxt SSR访客公开站点，服务端渲染
engine5173Vue3 SPA运营后台 + 可视化设计器 /designer
bff3100Next.js聚合 / 鉴权 / 转码
backend8080Spring Boot业务后端（唯一权威）

Go 版后端（luban-backend-go）已废弃，本仓不再纳入。完整架构与数据流见 整体架构。

核心能力

  • 可视化设计器：engine 的 /designer 全屏页，左物料面板 / 中画布 / 右属性面板，拖拽即搭页。完整操作流程见 产品导览。
  • 双渲染引擎：设计态 DesignRenderer（响应拖拽、选中、属性编辑）与运行态 RuntimeRenderer（只渲染），同一套 Schema 两种解释。
  • Schema 体系：物料协议 + 版本治理，是设计器与渲染端共用的「单一数据源」。
  • luban-ui 组件库：Vue3 物料库，独立 Storybook 开发。详见 组件库。
  • AI 助手：图片→低代码页面，把设计稿直接转成可编辑 Schema。详见 AI 助手与模块全景。

快速开始

git clone --recurse-submodules <luban-workspace>
make clone-all          # 初始化所有 submodule
make install-deps       # 各包装依赖（pnpm / mvn）
make dev                # 本地拉起 4 服务

默认端口见上方「4 应用服务」表。中间件（MySQL :13306 / Redis :16379）连远端 dev 服务器，无需本地起。

这不是教程

OpenLuban 不是「照着抄一遍」的项目。我写它，是把一套真正扛过大促的低代码架构用开源方式重新落地——Schema 怎么设计才不腐烂、双渲染怎么分、AI 怎么接进设计器循环。后面的子页只讲这些关键决策。
