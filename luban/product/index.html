产品导览

从登录到上线一个活动页，这是 OpenLuban 的完整用户旅程。重点看页面设计器——它是整个低代码平台的核心，也是 OpenLuban 区别于普通后台的关键。

注：截图取自真实运行环境（admin 视角），数据为开发联调样例。

登录与工作台

登录后第一眼是工作台 dashboard，顶部 4 张 KPI 卡（站点数 / 活动数 / 访问量 / 线索数）给运营「一眼看清大盘」。

左侧导航是整个运营后台的骨架：工作台、站点管理、页面、线索中心、表单管理、用户、系统设置。这套信息架构刻意「按运营动线」组织——运营 90% 的时间在「站点 → 页面 → 线索」这条主线上。

站点管理

站点是活动的容器。一个站点对应一个对外发布域，下面挂页面、表单、线索。

站点详情把一个站点的所有子资源（页面、表单、线索）聚合到一屏，运营不用在多级菜单里跳。这是个反直觉但好用的设计——按「站点」而非「资源类型」聚合，更贴合运营心智。

页面设计器（核心）

设计器是三栏布局：左物料面板、中画布、右属性面板。这是整个 OpenLuban 的灵魂。

核心交互闭环：左侧拖物料入画布 → 画布选中节点 → 右侧属性面板改 props → 画布实时预览。属性面板本质是物料 propsSchema 自动生成的表单——运营改的每一个值，都在改背后那棵 Schema 树。这正是「所见即所发」的底层机制：设计器产出的不是截图，是结构化 Schema。

深入设计器的架构与双渲染管线，见 整体架构。

表单编辑器

表单编辑器复用了设计器的三栏范式，但物料换成了表单字段（单行文本、下拉、单选、手机号等）。

一份表单 = 一棵字段 Schema，提交后落进线索中心。所以「表单」和「线索」是同一套数据的两面：表单管采集，线索管流转。

线索中心

线索中心是营销闭环的收口。表单收集到的线索在这里列表、筛选、流转。

线索按状态（新建 / 跟进中 / 已转化 / 已流失）流转，支持按来源、时间、状态多维筛选。低代码搭出的活动页把流量引进来，线索中心把流量沉淀成可运营的数据——这才是「低代码 + 营销」的完整价值链，而不是止步于「能搭页面」。

系统管理

用户管理做 RBAC：admin 才能进用户管理与系统设置，普通运营只能在自己授权的站点内操作（路由守卫 requiresAdmin 拦截）。多租户隔离在数据层完成，非 admin 用户仅能访问已授权站点。
