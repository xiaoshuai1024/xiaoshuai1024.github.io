组件库

luban-ui 是 OpenLuban 的 Vue3 组件库，既是设计器物料的载体，也可独立使用。它用 Storybook 做组件文档与隔离开发——每个物料一个 story，改组件先在 Storybook 里验证，再进设计器。

可交互组件预览

打不开？在新窗口查看完整 Storybook。iframe 固定高度 640px，建议新窗口全屏体验。

构建链路：luban-ui build-storybook → scripts/build-storybook.sh 拷贝到 static/luban-ui/ → Hugo 随博客一并部署，deploy.sh 零改动。更新物料后重跑该脚本即可。

物料 12 大类

设计器左侧物料面板按类目分组。当前注册 24 个物料，分布在 12 个类目：

类目用途典型物料
general通用基础LubanButton
content内容展示LubanBanner、LubanText、内容列表
layout布局容器Container、SidePanel
form表单字段输入框、下拉、单选
data-display数据展示Table
navigation导航Menu、Tabs
feedback反馈Modal、Drawer、Toast
lead线索线索表单组件
marketing营销营销活动物料
poster海报海报物料
website站点站点级物料

类目是物料的「目录」，不是技术分层。新增物料时先想清楚它属于哪个运营心智类目，再落地。

物料 Schema 协议

一个物料 = defineMaterial({ name, version, category, component, propsSchema })。以 LubanButton 为例：

export const buttonMaterial = defineMaterial({
  name: 'LubanButton',
  version: '1.0.0',
  category: 'general',
  description: '通用按钮，支持实心/描边/文本三种变体',
  component: LubanButton,
  propsSchema: {
    type: 'object',
    properties: {
      content: { type: 'string', default: '', label: '文案' },
      variant: {
        type: 'string',
        enum: ['contained', 'outlined', 'text'],
        default: 'contained',
        label: '变体',
      },
      color: {
        type: 'string',
        enum: ['primary', 'secondary', 'surface'],
        default: 'primary',
        label: '颜色',
      },
    },
  },
});

关键点：propsSchema 既是组件 prop 的真值来源，也是设计器右侧属性面板的表单定义。运营在属性面板改的每一个值，本质是在改这棵 propsSchema 描述的结构。这就是「属性面板 = Schema 表单」的底层机制——物料作者声明一次 propsSchema，设计器自动生成编辑 UI，无需手写属性面板。

核心 5 物料（已有 Storybook story）

这 5 个物料是 luban-base 里最先落地、配套 Storybook story 的，是组件库的「样板间」：

  • LubanButton — 三变体（contained/outlined/text）× 三色（primary/secondary/surface）
  • Form — 表单容器，聚合字段校验与提交
  • Layout — 页面布局骨架
  • LubanText — 富文本展示
  • LubanBanner — 顶部横幅

技术栈

选型
框架Vue 3 + Vite
文档/隔离开发Storybook（vue3-vite builder）
单仓组织pnpm workspace（luban-base 基础组件 + luban-low-code 物料）
物料注册materialRegistry + barrel 收口（24 物料统一注册）

物料分两层：luban-base（纯展示组件，无低代码耦合）和 luban-low-code（物料定义层，把 base 组件包成设计器可用的物料）。这个分层是刻意——base 组件能脱离低代码独立用在任何 Vue3 项目里，复用面更广。
