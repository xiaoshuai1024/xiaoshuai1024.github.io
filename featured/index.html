精选

这个博客讲一个完整的故事:我从前端出发,成长为全栈架构师,再转型为 AI 开发实践者。

下面按这三个阶段,把最能代表我能力、成长和认知的文章串成一条线。第一次来,从这里读起。

前端阶段:把界面工程做成体系

从一线前端到能独立设计前端架构,这段的核心是工程化思维——把零散的页面开发,沉淀成可复用的体系。

  • 复杂前端组件库架构 — 从单包到企业级 Monorepo 的组件库设计
  • 复杂中后台系统微前端(qiankun)接入实践 — 收拢技术栈各异的多个老系统
  • 基于 Nuxt 的 SSR 性能优化 — 把首屏从 1.8s 压到 0.5s
  • 前端团队基建体系建设 — 标准化、自动化、可复用
  • 从前端工程师到架构师 — 视角的升级:从执行到创造

全栈阶段:打通前后端的完整链路

做大厂架构,逼着我从「前端专家」长成「全栈架构师」。这段的核心是打通——从前端到 BFF、到 Java 后端、到数据库、到稳定性,独立兜住一条完整链路。

  • 企业级前端架构设计与落地实践 — Vue + Node + Java 一体化架构
  • 高并发低代码平台架构设计 — 架构经验源自大厂内部系统(峰值 10 万+ QPS)，开源实现见 OpenLuban
  • 技术驱动业务增长 — 架构如何赋能业务
  • 从零搭建企业级低代码平台全栈指南 — 六库分工的完整闭环
  • DDD 领域模型在低代码后端中的应用 — 限界上下文与统一契约
  • 低代码渲染引擎与设计器:从 schema 到可编辑画布 — 自研引擎的 schema/递归渲染/双拖拽
  • uni-app + SaaS 多租户小程序:从按租户打包到代开 — 一份模板 + ext_json 注入
  • 百万行 Vue2→Vue3 升级:babel codemod — 计划/预检/执行的事务化设计
  • 基于 SSR + CDN 的高并发全栈方案 — 吞吐量 200 → 2 万+
  • Web 稳定性监控体系实践 — 全链路、多维度、可量化
  • 多租户 SaaS 的数据隔离 — 从一行漏掉的 tenant_id 说起
  • RBAC 权限体系与过滤器链 — 注解声明 + 统一校验
  • Flyway 数据库迁移治理 — 版本号、squash 和那些启动失败
  • 可观测性:RequestId 与监控告警 — 串成一条能排查的链路
  • 给 SaaS 做 2.5 万用户压测 — 用数据验证架构假设

AI 开发阶段:用专家的方式驾驭 AI

最后这一段,我把重心放到 AI 开发。主力工具是 Claude Code、Cursor、OpenCode。核心是把 AI 协同研发做成工程体系——不是「尝试用 AI」,而是已经把它做成可复用的方法论。

这条线分三层:认知与方法(为什么转、怎么转)→ 真实账本(4 人 + AI 干出什么)→ 工程体系(架构、质量、规范怎么兜)。

认知与方法

  • AI 时代研发认知转变:知识不再是护城河,老兵反而成了红利 — 当 AI 全知全能,工程师还靠什么吃饭
  • AI 时代,什么能力对研发才是真正重要的 — 不是"会写代码",是"知道如何正确地做事"
  • AI 时代研发转型之我见 — 从前后端到全栈,再到产品开发专家

真实账本(实战系列)

  • 4 个人 + AI,2 个月交付近百万行:一份真实的研发账本 — 总账:老兵 + AI 杠杆被拉满的真实数据
  • 微信分润从 0 到 1:CPS 分账、提现状态机与一次冲正降级 — 资金链路最硬的一块
  • 850 张表 + 400 个迁移:多租户 SaaS 的 Flyway 治理实战 — squash、版本号演进、生产 gap 止血
  • 数据大屏背后:多租户 SaaS 的统计中台怎么搭 — T+1 汇总表 + Redis + ES 三层架构
  • 把"用 AI"做成工程体系:28 个命令和一条自治评审回路 — 这套产能的真正根

工程体系(架构与质量兜底)

  • AI 编程之我见 — 序章:AI 怎样走进我的日常
  • 大型全栈项目架构设计 + AI 保障 — 把「AI 保障」建进系统骨架
  • 我们的 AI 研发工作流 — 从原型到上线的全链路
  • 没有 CI,用多 Agent 评审回路兜住代码质量 — 自治评审 + 双计数器收敛
  • 编译能过 ≠ 代码正确 — 一次「幽灵删除」事故复盘
  • 提交前的四道契约闸 — 守住 API/SQL/权限/迁移边界
  • UT/IT/E2E 三层测试纪律 — fail-fast、禁跳过、零占位
  • 覆盖率门禁的「说」与「做」 — 从墙上的 95% 到配置里的 25%
  • 把项目规范写进代码底层 — 让新人和 Agent 开机即用
  • 事故即规范 — 把事故固化成机器可执行的规则

AI 应用集成(多租户 RAG + Agent)

  • 构建一个多租户 AI 助手:Spring AI + RAG + Agent — 多租户 AI 应用的整体架构
  • RAG 知识库从 0 到 1 — 多租户检索增强
  • Agent / Tool Calling 桥接设计 — 工具定义与框架解耦
  • 多租户 AI 配置:动态 ChatModel 工厂 — 模型实例动态化
